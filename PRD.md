# AI Ukulele Academy PRD

版本：v0.1  
日期：2026-07-01  
项目定位：以“AI 音乐学园”的公开功能形态为参考，优先开发面向尤克里里弹唱练习的 AI 互动教学 App。

> 合规边界：本产品借鉴公开可见的功能模式和学习体验，不复制“AI 音乐学园”的受版权保护内容、课程视频、曲谱资源、UI 素材、品牌名称、文案和会员内容。

## 0. 本次整合结论

本版文档已合并两套 PRD/TDD/SDD 的精华：保留当前文档中更完整的尤克里里聚焦、算法评测、数据模型和本地优先架构；吸收外部文档中更清晰的 P0-P3 优先级、歌曲定制、游戏化、内容后台、商业化路径和首批开发任务。

最终方向确认如下：

- 第一阶段不做“大而全音乐学园”，先做真实可用的尤克里里弹唱练习闭环。
- 产品体验优先级为：调音稳定 > 跟练反馈可信 > 练习报告有用 > 课程和曲谱丰富。
- 拾音策略不押单一算法，MVP 用 MPM/YIN/YINFFT 类轻量 DSP，后续用 CREPE/Basic Pitch/SPICE 等模型增强复杂场景。
- 页面设计采用设计系统思路推进：移动端偏工具型、低干扰、高可读；Web 管理后台/Agent 控制台可采用 shadcn/ui + Tailwind 风格。
- 开发过程引入多 Agent 协作，但每个 Agent 必须产出可验收 artifact，不能只输出建议。

## 1. 背景与机会

用户已经使用过“AI 音乐学园”进行基础弹唱练习，希望进入更高阶训练，但会员成本较高。因此本项目目标是开发一款自有 App，逐步实现类似的 AI 互动音乐学习能力，并在后期扩展更多自定义内容。

公开资料显示，“AI 音乐学园”定位为 AI 智能互动音乐教学 App，覆盖吉他、钢琴、尤克里里、声乐等乐器，核心特征包括乐音识别、AI 互动视频教学、实时纠错、个性化学习、调音器、节拍器、和弦库、节奏练习、和弦练习、曲谱库和课程分层。

本项目一期不追求覆盖所有乐器，先聚焦“尤克里里弹唱”的完整闭环：调音、节拍、和弦、节奏、弹唱跟练、实时反馈、练习记录。

## 2. 目标用户

### 2.1 核心用户

- 零基础到进阶初级的尤克里里学习者。
- 已经会基础和弦，希望提升节奏、换和弦、弹唱同步能力的用户。
- 希望用手机完成自学、跟练、反馈和复盘的用户。

### 2.2 次级用户

- 吉他/钢琴等其他乐器学习者，后期可复用课程、拾音、节奏和曲谱模块扩展。
- 自建课程或自定义训练内容的创作者。

## 3. 产品目标

### 3.1 短期目标

- 建立可用的尤克里里 AI 互动练习 MVP。
- 实现稳定调音器、节拍器、和弦库、基础跟练和练习评分。
- 建立拾音算法评测体系，避免后续开发被算法误判拖住。
- 建立多智能体规划平台，把复杂功能拆分、排期、验收和整合。

### 3.2 中期目标

- 实现课程化学习路径：基础课、专项课、歌曲课、高阶课。
- 支持弹唱练习：和弦切换、节奏型、歌词/小节进度、录音回放、AI 评价。
- 支持自定义曲谱导入和练习计划。

### 3.3 长期目标

- 接近“AI 音乐学园”公开功能模块的完整覆盖。
- 扩展到吉他、声乐、钢琴等更多乐器。
- 建立内容生产、课程发布、社区分享、付费订阅能力。

## 5. MVP 范围

### 5.1 一期必做

1. 用户无需登录即可体验基础工具。
2. 尤克里里调音器：G-C-E-A 标准调弦，实时显示音名、偏差 cents、稳定度。
3. 节拍器：BPM、拍号、重音、声音开关、振动开关。
4. 和弦库：基础和弦指法图、难度、常见替代指法。
5. 基础训练：单音识别、和弦切换、节奏跟练。
6. 歌曲跟练样例：曲谱进度、歌词/和弦提示、录音回放。
7. 练习评分：音准、节奏、连续性、完成度、建议。
8. 本地练习记录：日期、时长、分数、薄弱项。
9. 算法评测台：导入录音样本，比较 MPM/YIN/CREPE/SPICE/其他候选结果。
10. 多智能体规划面板：任务拆分、子任务状态、验收清单、整合结果。

### 5.2 一期不做

- 不做完整付费会员系统。
- 不做大规模曲谱版权库。
- 不做直播课、社区、老师一对一。
- 不做钢琴/吉他完整课程。
- 不承诺复杂和弦的实时复音精确识别。

### 5.3 P0-P3 优先级分层

P0 MVP：

- 游客模式、本地档案、乐器选择、学习目标。
- 麦克风授权、输入电平、环境噪声检测、耳机/外放提示。
- 尤克里里调音器、节拍器、基础课程播放器。
- 单音/分解和弦跟练、简单节奏判定、练习报告。
- 基础闯关机制：通过阈值、失败复练、成绩记录。

P1 核心闭环：

- C、Am、F、G、G7 等常见和弦练习。
- 弹唱练习：伴奏播放、歌词/和弦同步、扫弦节奏判定。
- 曲谱视图：歌词、和弦、节奏型、小节线、滚动播放。
- 错误定位：错音、错和弦、节奏偏移、未弹、噪声干扰。
- 成就系统：连续打卡、阶段徽章、过关记录。

P2 高阶能力：

- 歌曲定制：导入授权音频/歌词/和弦，生成简化版弹唱练习。
- 难度自适应：根据历史准确率动态调整 BPM、和弦数量、节奏型。
- AI 练习教练：基于练习报告生成下一次练习计划。
- 多乐器扩展和离线 ML 复盘。

P3 商业化与运营：

- 会员、课程包、单曲包、虚拟币或权益系统。
- 内容后台：课程、曲谱、练习模板、评分规则配置。
- 活动运营：限时挑战、榜单、训练营、优惠券。

## 6. 核心用户旅程

### 6.1 新手练习旅程

1. 打开 App，选择“尤克里里”。
2. 使用调音器完成 G-C-E-A 调音。
3. 进入基础课程，学习 C、Am、F、G。
4. 跟随节拍器做 60 BPM 慢速和弦切换。
5. App 实时提示弹错、慢拍、快拍或未检测到有效声音。
6. 练完后看到评分、错点和下次建议。

### 6.2 弹唱练习旅程

1. 选择歌曲样例或导入曲谱。
2. App 展示小节、和弦、歌词和节奏型。
3. 用户跟随伴奏/节拍弹唱。
4. App 记录音频并做实时/准实时反馈。
5. 结束后输出音准、节奏稳定性、和弦切换点、弹唱同步建议。

### 6.3 高阶自定义旅程

1. 用户创建专项训练：例如 F 和弦、切音、16 分节奏。
2. 设置 BPM、循环次数、目标准确率。
3. 系统跟踪连续多天表现，推荐升速或降速。

## 7. 功能需求

### 7.1 调音器

- 支持尤克里里标准调弦 G4/C4/E4/A4。
- 显示当前检测频率、目标频率、音名、cents 偏差。
- 显示稳定度和置信度。
- 支持自动识别当前弦，也支持手动选弦。
- 连续稳定命中目标范围后显示完成状态。
- 默认可接受范围：±5 cents 精准，±10 cents 可接受。

### 7.2 节拍器

- BPM 范围：30-240。
- 拍号：2/4、3/4、4/4、6/8。
- 重音开关、细分音开关。
- 可用于练习模块同步时间轴。

### 7.3 和弦库

- 基础和弦：C、Am、F、G、G7、Dm、Em、A7、D7 等。
- 显示指板图、按弦手指、空弦/闷弦。
- 支持收藏和最近练习。
- 后期支持转调、和弦替换、常见进行。

### 7.4 互动课程

- 课程由 lesson script 驱动，包含视频/音频/文本/练习节点。
- 节点类型：讲解、示范、跟练、测验、复盘。
- 跟练节点可绑定目标音符、目标和弦、节奏型、BPM。
- 支持重新练习、慢速练习、循环片段。

### 7.5 曲谱与弹唱

- 一期使用自有样例曲谱和用户本地自定义曲谱。
- 曲谱结构包含：标题、调式、速度、拍号、小节、和弦、歌词、节奏型。
- 练习时按时间轴滚动，支持暂停、回退、循环小节。
- 后期支持 MusicXML、MIDI、ChordPro 或自定义 JSON 导入。

### 7.6 实时反馈与评分

- 实时反馈：当前音高、节奏偏差、漏弹、过早/过晚。
- 练后评分：音准、节奏、完成度、稳定性、难点。
- 不直接用单一总分遮蔽问题，必须给出可操作建议。
- 支持录音回放，并在时间轴标注错误点。

### 7.7 多智能体规划平台

- 用户输入大目标，例如“实现调音器”或“开发歌曲跟练”。
- 系统自动拆分为产品、算法、前端、后端、测试、文档子任务。
- 每个子任务包含目标、输入、输出、依赖、验收标准。
- 支持多个 Agent 并行执行，最终由集成 Agent 汇总。
- 所有 Agent 产出必须可追踪、可回滚、可验收。

### 7.8 页面体验与设计系统

页面设计目标不是做营销式首页，而是做“练习效率高、反馈可信、长期使用不累”的音乐训练工具。

移动端设计原则：

- 首页直接进入练习和工具，不做空泛 landing page。
- 调音器、节拍器、跟练页必须信息密度适中，关键反馈足够大，弱信息退后。
- 调音状态用音名、cents、稳定度、颜色和动效共同表达，但动效不能影响判断。
- 练习页面避免装饰性卡片堆叠，优先时间轴、曲谱、反馈和控制区。
- 触控目标不小于 44x44px，图标按钮必须有可理解语义。
- 支持浅色/深色模式，保证弱光练习和夜间练习可用。
- 颜色要区分功能：命中、偏高、偏低、漏弹、噪声、等待输入，不使用单一色系硬撑全局。

设计系统要求：

- 建立三层 token：primitive、semantic、component。
- 统一颜色、字体、间距、圆角、阴影、动效时长。
- 关键组件需定义默认、按下、禁用、错误、成功、加载等状态。
- Web 管理后台和 Agent 控制台可使用 shadcn/ui + Tailwind CSS；移动端使用同一套 token 映射到 React Native 组件。
- `ui-ux-pro-max` 目前本地仅有目录入口，完整上游模板库未安装；当前先使用 `ckm:ui-styling` 与 `ckm:design-system` 的设计原则。

## 8. 非功能需求

### 8.1 拾音体验

- 调音器端到端反馈延迟目标：小于 80 ms，最大不超过 150 ms。
- 跟练实时提示延迟目标：小于 120 ms，最大不超过 250 ms。
- 普通室内噪声下保持稳定。
- 需要静音/噪声门限，避免环境音误触发。

### 8.2 性能

- App 冷启动：目标小于 2.5 秒。
- 练习页面帧率：目标 60 FPS。
- 音频处理不得阻塞 UI。
- 算法模块必须支持采样率、窗口长度和 hop size 配置。

### 8.3 隐私

- 默认本地处理麦克风音频。
- 录音上传必须显式授权。
- 用户练习记录可本地删除。

### 8.4 可扩展性

- 乐器、调弦、课程、算法、评分规则都必须配置化。
- 算法模块使用统一接口，方便替换 MPM/YIN/CREPE/SPICE/SwiftF0 等实现。

## 10. 成功指标

- 调音器在样本集上的音名识别准确率大于 98%。
- 单音检测 cents 误差中位数小于 5 cents。
- 基础节奏跟练误差检测准确率大于 90%。
- MVP 用户完成首节课比例大于 70%。
- 用户完成一次调音 + 一次练习 + 一次复盘的闭环时间小于 10 分钟。

## 11. 主要风险

- 复音和弦识别难度高，不能只依赖单音 pitch detection。
- 弹唱场景包含人声、扫弦、噪声，实时识别会比调音器复杂。
- 曲谱和课程内容涉及版权，需要自建、授权或用户本地导入。
- 深度学习算法可能精度高但端侧性能、延迟、包体积和电量成本较高。
- 多智能体平台若缺少验收机制，会变成任务列表而非协作系统。
