# 练琴入口页开发文档

版本：v0.1
日期：2026-07-12
页面：Practice Hub / 练琴入口页
阶段：先完成 Preview，再同步 App

## 1. 页面目标

练琴入口页是“工具与专项练习”的总入口，不直接承载完整跟练流程。

用户进入底部 Tab 的“练琴”后，应能一屏看到：

- 智能调音器。
- 节拍器。
- 和弦库。
- 节奏练习。
- 和弦转换。
- 今日建议路径。

页面作用是分发，不是堆练习报告，也不是直接进入 C-Am-F-G7 跟练。

## 2. 用户场景

### 2.1 练习前准备

用户目标：

- 先调音。
- 开节拍器熟悉速度。
- 查看不会按的和弦。

页面响应：

- 调音器、节拍器、和弦库放在工具区。
- 入口明确可点。

### 2.2 专项练习

用户目标：

- 练节奏型。
- 练和弦转换。
- 进入后续跟弹训练。

页面响应：

- 节奏练习和和弦转换放在专项练习区。
- 显示今日建议路径，让新手知道顺序。

## 3. 页面结构

首屏从上到下：

```text
顶部
  标题：练琴
  副标题：调好琴、找好拍、看好指法再开始

工具区
  智能调音器
  节拍器
  和弦库

专项练习区
  节奏练习
  和弦转换

今日建议路径
  1 调准四根弦
  2 60 BPM 下扫四拍
  3 C-Am 转换
```

布局要求：

- 一屏优先。
- 不展示长练习历史。
- 不展示完整课程路径。
- 入口卡片要比首页更偏工具型。

## 4. 模块规格

### 4.1 顶部区域

内容：

- 标题：练琴。
- 副标题：调好琴、找好拍、看好指法再开始。
- 可选状态：今日目标 15 分钟 / 本地练习。

验收：

- 用户知道这是练琴工具页，不是课程页。

### 4.2 工具区

三个入口：

| 入口 | 说明 | 目标 |
| --- | --- | --- |
| 智能调音器 | 标准 GCEA，拨弦识别 | 调音器页 |
| 节拍器 | BPM、重音、声音 | 节拍器页 |
| 和弦库 | 指法图、试听 | 和弦库页 |

验收：

- 三个工具入口可点击。
- 调音器入口视觉权重可略高。
- 和弦库入口必须强调“指法图”。

### 4.3 专项练习区

两个入口：

| 入口 | 说明 | 目标 |
| --- | --- | --- |
| 节奏练习 | 下扫四拍、下下上上 | 节奏练习页 |
| 和弦转换 | C-Am、Am-F、F-G7 | 和弦转换页 |

验收：

- 入口卡片能说明练什么。
- 后续进入独立页面，不在入口页展开完整练习。

### 4.4 今日建议路径

内容：

- 3 个步骤。
- 每步短标题 + 一句说明。
- 可点击的主操作：开始第一步。

建议默认：

- 调准四根弦：G-C-E-A 全部进入绿色区。
- 节奏型练习：60 BPM 下扫四拍，稳定 2 轮。
- 和弦转换：C-Am 先练两个和弦。

验收：

- 新手能理解练习顺序。
- 不把历史记录塞到这里。

## 5. 数据来源

Preview 阶段：

- 可在 `preview.html` 内使用常量。
- 复用现有 sheet：`tuner`、`metronome`、`chords`、`rhythm`、`transition`。

App 同步阶段：

- 底部 `练琴` Tab 应渲染 Practice Hub，而不是直接渲染 PracticeScreen。
- 原 PracticeScreen 保留为内部练习运行页。
- 首页的“练习”入口进入 Practice Hub。
- 今日推荐的“按建议开始”可继续进入 PracticeScreen。

建议路由：

```ts
type Tab = "home" | "practice" | "songs" | "learn" | "me" | "tuner" | "chords" | "practiceRunner";
```

## 6. 视觉要求

整体：

- 工具型、紧凑、清晰。
- 比首页更直接，不做营销感。
- 卡片高度低，适合一屏。

推荐布局：

- 顶部短标题。
- 智能调音器使用横向大卡。
- 节拍器 / 和弦库使用双列小卡。
- 节奏练习 / 和弦转换使用双列小卡。
- 今日建议路径使用紧凑三步列表。

## 7. Preview 实现要求

## 9. 验收标准

练琴入口页完成必须满足：

- 底部点击“练琴”后，不再直接进入跟练流程。
- 页面显示智能调音器、节拍器、和弦库、节奏练习、和弦转换。
- 所有入口可点击并有反馈。
- 页面核心内容一屏可见。
- 不出现长练习报告和历史记录。
- Preview 与 App 的入口含义一致。
