# 和弦转换页开发文档

页面：Chord Transition / 和弦转换页
状态：开发中

## 1. 页面目标

和弦转换页负责让新手把“会按单个和弦”推进到“能按节拍换到下一个和弦”。

本页不是和弦库，也不是歌曲跟弹页。它只解决一个核心问题：用户能否在下一小节第一拍前，把左手从当前和弦稳定换到目标和弦。

用户进入页面后，应能完成：
- 选择一组常用和弦转换。
- 同时看到起始和弦与目标和弦的指法图。
- 跟随 1 / 2 / 3 / 4 拍练习换指。
- 在第 4 拍执行换和弦动作。
- 获得本地模拟的转换稳定度评分。
- 完成后写入本地练习记录。

## 2. 用户场景

### 2.1 刚学会 C 和 Am

用户已经知道 C 和 Am 怎么按，但换指容易慢。

页面响应：
- 默认进入 C -> Am。
- 左侧显示 C 指法图，右侧显示 Am 指法图。
- 第 1-3 拍提示“稳住 / 保持 / 提前准备”。
- 第 4 拍提示“换到 Am”。

### 2.2 从节奏练习进入和弦转换

用户已经能跟着右手节奏扫弦，需要加入左手。

页面响应：
- 使用 60 BPM 起步。
- 不请求麦克风权限。
- 点击开始后自动推进节拍、记录换指尝试次数。

### 2.3 准备进入歌曲片段

用户准备弹 C-Am-F-G7 的入门歌曲片段。

页面响应：
- 支持 C -> Am、Am -> F、F -> G7、G7 -> C，以及 C/G 调常用转换。
- 完成一组后可手动切换下一组。
- 本阶段允许 4 次尝试后自动进入下一组，帮助形成循环感。

## 4. 功能规格

### 4.1 转换组

MVP 至少支持 12 组常用转换：

| 转换 | 目标 | 默认 BPM |
| --- | --- | ---: |
| C -> Am | 从单指 C 切到 Am | 60 |
| Am -> F | 保留二指，加一指 | 60 |
| F -> G7 | 提前三指落位 | 60 |
| G7 -> C | 收回到单指 C | 60 |
| C -> G | 从单指 C 进入 G 的三指手型 | 60 |
| G -> Am | 从 G 收回到 Am | 60 |
| Am -> G | 从 Am 展开到 G 三角手型 | 60 |
| C -> F | 从 C 加到 F 的双指手型 | 60 |
| G -> D | G 调常见转换 | 60 |
| D -> Em | G 调常见转换 | 60 |
| Em -> C | G 调常见转换 | 60 |
| D -> G | G 调常见转换 | 60 |

切换转换组时：
- 暂停当前定时器。
- 当前拍归零。
- 本组尝试次数归零。
- 本组评分归零。

### 4.2 指法图

本页所有和弦必须显示指法图，不允许只显示 C、Am、F、G7 英文代码。

指法图要求：
- 复用和弦库的 G / C / E / A 四弦顺序。
- 显示顶部 ○ / ● / ×。
- 显示按弦圆点和手指编号。
- 当前需要关注的和弦高亮。

### 4.3 四拍练习

默认每 4 拍完成一次换指尝试：

| 拍号 | 提示 |
| --- | --- |
| 1 | 稳住起始和弦 |
| 2 | 保持手型 |
| 3 | 提前准备目标手指 |
| 4 | 换到目标和弦 |

第 4 拍时记录一次换指尝试。

### 4.4 评分与反馈

MVP 阶段使用本地模拟评分：
- 点击开始后自动推进节拍。
- 第 4 拍自动记录一次换指尝试。
- 每次尝试生成一个稳定度分数。
- 页面显示“准时 / 偏早 / 偏晚”类型反馈。
- 完成本组后写入本地练习记录。

后续真实音频阶段：
- 用扫弦 onset 与换指完成事件判断换指是否落在目标拍前。
- 结合 PitchFrame / chord detection 评估目标和弦是否按对。
- 不在本阶段接真实麦克风评分。

## 8. 验收标准

- 从练琴入口点击“和弦转换”能打开页面。
- 页面显示 C -> Am、Am -> F、F -> G7、G7 -> C、C -> G、G -> Am、Am -> G、C -> F、G -> D、D -> Em、Em -> C、D -> G。
- 每组都显示起始和弦、目标和弦指法图。
- 点击开始后，四拍自动推进。
- 第 4 拍自动增加一次尝试次数。
- 稳定度评分会随尝试更新。
- 点击完成本组后，最近练习记录更新。
- 页面不请求麦克风权限。
- 375px 和 430px 宽度下核心内容不明显溢出。
