# 节拍器页开发文档

版本：v0.1
日期：2026-07-13
页面：Metronome / 节拍器
阶段：先完成 Preview，再同步 App

## 1. 页面目标

节拍器页负责帮助新手建立稳定节奏感，是后续节奏型练习、和弦转换和歌曲跟弹的基础工具。

本页不做复杂评分，也不替代节奏练习页。它只提供一个清楚、可听、可看的基础节拍器。

用户进入页面后，应能一屏内完成：

- 选择常用速度：慢速 60 / 标准 70 / 进阶 85。
- 使用 -5 / +5 微调 BPM。
- 点击开始 / 暂停。
- 看到当前第几拍。
- 区分第一拍重音和后续轻音。
- 打开 / 关闭节拍声。
- 看到已运行小节数。

## 2. 用户场景

### 2.1 新手找拍

用户目标：

- 先用慢速 60 BPM 熟悉四拍循环。
- 看着第一拍重音和后三拍轻音练右手。
- 不需要立刻进入评分。

页面响应：

- 默认 70 BPM，但提供慢速 60 的一键预设。
- 当前拍用大小、颜色和文字同时提示。
- 第一拍用红色重音点，后续轻拍用蓝色点。

### 2.2 跟练前准备

用户目标：

- 在正式节奏型或和弦转换前，先听几小节。
- 知道当前声音是否打开。
- 快速进入节奏型练习。

页面响应：

- 节拍声开关必须可见。
- 当前平台没有真实音频时，Preview 仍应使用 WebAudio click 声。
- 页面底部提供“去练节奏型”入口。

## 3. 页面结构

首屏从上到下：

```text
顶部
  返回
  节拍器
  4/4

速度区
  当前 BPM
  每拍毫秒
  慢速 60 / 标准 70 / 进阶 85

节拍可视化
  1 2 3 4 四个圆点
  第一拍重音红点
  后三拍轻音蓝点
  当前拍放大或描边

控制区
  -5
  开始 / 暂停
  +5
  节拍声 开 / 关

状态与入口
  已运行小节
  声音状态
  去练节奏型
```

布局要求：

- 375px 宽度下核心内容一屏可见。
- 节拍点是页面视觉中心。
- 不使用长篇说明。
- 控制按钮必须足够大，适合手机触控。

## 4. 模块规格

### 4.1 顶部区域

内容：

- 标题：节拍器。
- 右侧：4/4。
- 返回：回到练琴入口页。

验收：

- 用户明确知道当前是独立节拍器页。
- 返回可回到练琴入口页。

### 4.2 速度区

内容：

- 当前 BPM 大号显示。
- 每拍毫秒小字显示。
- 三个预设：慢速 60、标准 70、进阶 85。

交互：

- 点击预设立即切换 BPM。
- 如果节拍器正在运行，切换 BPM 后节拍间隔立即更新。
- -5 / +5 范围：40 到 180 BPM。

验收：

- 当前预设有高亮。
- 自定义 BPM 时三个预设都不高亮，或显示“自定义”状态。

### 4.3 节拍可视化

内容：

- 4 个节拍点。
- 第 1 拍为重音。
- 第 2/3/4 拍为轻音。
- 当前拍放大或描边。

验收：

- 不只依赖颜色区分，还要有大小、描边或文字变化。
- 停止时停留在第 1 拍。

### 4.4 声音开关

内容：

- 节拍声 开 / 关。
- 声音状态文案。

交互：

- 开启时播放 click 声。
- 第一拍使用更高或更重的声音。
- 关闭时只显示视觉节拍。

验收：

- Preview 阶段使用 WebAudio。
- App 阶段先复用当前平台可用能力；如原生声音未接，必须显示“当前平台先显示节拍”的状态，不伪装成功。

### 4.5 状态与入口

内容：

- 已运行小节数。
- 当前速度建议。
- “去练节奏型”按钮。

验收：

- 小节数每 4 拍加 1。
- 点击“去练节奏型”进入节奏型练习页或对应 sheet。

## 5. 数据来源

Preview 阶段：

- 可在 `preview.html` 内使用本页状态：
  - `bpm`
  - `beat`
  - `barCount`
  - `soundEnabled`
- 复用已有 `playClick(accent)`。

App 阶段：

- 默认 BPM 使用 `chordLoopPractice.bpm`。
- 速度预设可复用 `practiceTempoPresets`。
- 后续可抽到 shared，但本页阶段先保持页面内状态。

## 6. Preview 实现要求

文件：

- `apps/mobile/dist-web/preview.html`

必须完成：

- 节拍器 sheet 使用新结构。
- BPM、预设、四拍点、开始/暂停、-5/+5 可用。
- 节拍声开关可用。
- 第一拍重音和轻拍视觉区分。
- 运行时每拍推进，4 拍后小节数加 1。
- “去练节奏型”能打开节奏练习 sheet。

暂不要求：

- 复杂拍号，如 3/4、6/8。
- 节奏评分。
- 保存节拍器使用记录。

## 7. App 同步要求

文件：

- `apps/mobile/App.tsx`

必须完成：

- `MetronomeScreen` 与 Preview 页面结构一致。
- 预设 BPM、-5/+5、开始/暂停可用。
- 四拍点显示重音和轻拍。
- 声音开关有状态反馈。
- 小节数随节拍推进。
- 可进入节奏型练习。

暂不要求：

- 原生音频 click 声完整接入。
- 后台播放。
- 复杂节拍编辑器。

## 8. 验收标准

节拍器页完成必须满足：

- 从练琴入口页点击“节拍器”能进入。
- 页面显示当前 BPM、三档预设、四拍点、声音开关和控制按钮。
- 点击开始后当前拍按 BPM 推进。
- 第 1 拍和后续轻拍视觉上明显不同。
- 点击 -5 / +5 会改变 BPM。
- 点击慢速 / 标准 / 进阶会切换 BPM。
- 点击节拍声开关会改变状态。
- 点击“去练节奏型”进入节奏练习。
- Preview 与 App 都通过验证。

人工验证：

- 打开 `preview.html`。
- 进入“练琴”。
- 点击“节拍器”。
- 点击“开始”，确认 1/2/3/4 逐拍推进。
- 点击“慢速 60 / 标准 70 / 进阶 85”，确认 BPM 变化。
- 点击 -5 / +5，确认 BPM 变化。
- 点击“节拍声 开/关”，确认状态变化。
- 等待 4 拍，确认小节数增加。
- 点击“去练节奏型”，确认进入节奏练习页。

## 10. 暂不做

本页阶段暂不做：

- 3/4、6/8 和自定义拍号。
- 后台节拍器。
- 原生音频延迟优化。
- 节拍练习评分。
- 云端节拍器偏好保存。

这些进入节奏练习页、音频工程或设置页阶段处理。
