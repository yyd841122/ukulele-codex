# 和弦库页开发文档

版本：v0.1
日期：2026-07-13
页面：Chord Library / 和弦库
阶段：先完成 Preview，再同步 App

## 1. 页面目标

和弦库页负责让新手快速查找、看懂、练习尤克里里和弦。

本页必须以“指法图”为核心展示，而不是只显示 C、G、Am、F 这样的英文代码。所有和弦相关入口，包括歌曲卡、和弦转换、跟练目标，都应逐步复用同一套指法图组件。

用户进入页面后，应能完成：

- 搜索和弦。
- 按分类查看和弦。
- 直接看到每个和弦的指法图。
- 看到指法数字，如 `0-0-0-3`。
- 区分入门和弦与后续扩展和弦。
- 点击和弦获得反馈，后续可接试听、收藏、练习。

## 2. 用户场景

### 2.1 新手查和弦

用户目标：

- 不知道 C / G7 怎么按。
- 需要直接看图，不想只看代码。
- 想知道哪根弦空弦、哪根弦按几品。

页面响应：

- 每张和弦卡必须显示小指法图。
- 卡片下方显示和弦名和指法数字。
- 指法图使用 G / C / E / A 四根弦顺序。

### 2.2 从歌曲或练习跳转

用户目标：

- 在歌曲或和弦转换里看到某个和弦，想点进去看详细指法。
- 练完后能返回原练习路径。

页面响应：

- 本阶段至少保证入口一致：点击和弦有反馈。
- 后续可扩展为和弦详情 sheet。

## 3. 页面结构

首屏从上到下：

```text
顶部
  返回
  和弦大全
  已收录数量

搜索框
  搜索和弦，如 C、G7、Am

分类 Tabs
  所有
  入门和弦
  大三和弦
  小三和弦
  属七和弦

和弦网格
  每张卡片：
    小指法图
    和弦名
    指法数字
    试听/练习入口占位

收藏区
  收藏的和弦
  MVP 阶段可显示占位或本地收藏
```

布局要求：

- 手机宽度下使用 2 列或 3 列网格，优先保证指法图清晰。
- 卡片不要过高，避免首屏只看到两三个和弦。
- 搜索和分类必须固定在页面上方，方便快速查找。

## 4. 模块规格

### 4.1 顶部区域

内容：

- 标题：和弦大全。
- 右侧：已收录数量。
- 返回：回到练琴入口页。

验收：

- 用户知道当前是和弦库，不是练习页。

### 4.2 搜索框

内容：

- 占位：搜索和弦，例如 C、G7。

交互：

- 输入后过滤和弦名。
- 空结果显示清楚提示。

验收：

- 搜索 `C` 能看到 C / C7 等匹配。
- 搜索 `Am` 能看到 Am。

### 4.3 分类 Tabs

分类：

| 分类 | 说明 |
| --- | --- |
| 所有 | 全部可用和弦 |
| 入门和弦 | C、Am、F、G7 |
| 大三和弦 | C、G、D、F、E 等 |
| 小三和弦 | Am、Em、Bm 等 |
| 属七和弦 | G7、C7、D7 等 |

验收：

- 当前分类有高亮。
- 分类切换后网格更新。

### 4.4 和弦卡片

每张卡片必须包含：

- 小指法图。
- 和弦名。
- 指法数字。
- 播放/练习按钮占位。

指法图要求：

- 顶部显示 ○ / ● / ×。
- 指板内显示圆点。
- 圆点内显示手指编号。
- 底部显示 G / C / E / A。

验收：

- 不能只显示英文和弦名。
- C、Am、F、G7 必须正确：
  - C：`0-0-0-3`
  - Am：`2-0-0-0`
  - F：`2-0-1-0`
  - G7：`0-2-1-2`

### 4.5 收藏区

内容：

- 收藏的和弦标题。
- 本阶段默认显示 C、Am、F、G7 或空状态。

验收：

- 收藏区不影响主网格查找。
- 后续可接本地收藏。
