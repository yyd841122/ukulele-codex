# 节奏练习页开发文档

页面：Rhythm Practice / 节奏练习页

状态：待开发

## 1. 页面目标

节奏练习页负责让新手先把右手扫弦节奏练稳定，再进入和弦转换和歌曲弹唱。

本页不是普通节拍器。节拍器只提供稳定拍点；节奏练习页必须告诉用户“这一拍怎么扫”，并记录本组完成情况。

用户进入页面后，应能完成：

- 选择一个入门节奏型。
- 看懂每一拍的扫弦方向和强弱。
- 跟随可视化节拍点练习。
- 开始、暂停、完成本组、切换节奏型。
- 获得本地模拟评分和下一步建议。

## 2. 用户场景

### 2.1 第一次练右手节奏

用户刚学尤克里里，还不熟悉扫弦。

页面响应：

- 默认进入最简单的“下扫四拍”。
- 第一拍使用重音红点，其余轻拍使用蓝绿色点。
- 当前拍放大或高亮，不能只靠颜色区分。
- 当前动作显示为“下扫 / 轻扫 / 停 / 下上”等明确指令。

### 2.2 从节拍器进入节奏练习

用户在节拍器页听了几小节后，点击“去练节奏型”。

页面响应：

- 沿用节拍器的 BPM 或使用节奏型默认 BPM。
- 节拍声默认可开关。
- 进入后不请求麦克风权限。

### 2.3 为歌曲练习做准备

用户准备进入歌曲前，需要熟悉歌曲常用节奏。

页面响应：

- 可切换到“下下上上”等常见弹唱节奏型。
- 显示该节奏型适合的后续练习：和弦转换、歌曲片段。
- 完成后写入本地练习记录，用于首页和我的页展示。

## 4. 功能规格

### 4.1 节奏型选择

本阶段至少支持六个常用节奏型：

| 节奏型 | 默认 BPM | 拍点 | 说明 |
| --- | ---: | --- | --- |
| 下扫四拍 | 60 | 下 / 下 / 下 / 下 | 第一拍重音，后三拍轻扫 |
| 下下上上 | 60 | 下 / 下上 / 上 / 下上 | 常见弹唱扫弦方向 |
| 连续下上 | 60 | 下上循环 | 八分音符连续扫弦，练手腕稳定 |
| 二四切音 | 60 | 下 / 切 / 下 / 切 | 在第 2、4 拍加入切音 |
| 三拍子 | 60 | 下 / 下 / 上 | 强弱弱的 3/4 慢歌节奏 |
| 民谣切分 | 65 | 下 / 空 / 上 / 下 / 上 | 带空拍的常见民谣切分 |

切换节奏型时：

- 停止当前计时器。
- 当前拍归零。
- 小节计数保留或重置，本阶段先重置。
- BPM 切换为节奏型默认值。

### 4.2 节拍点

节拍点必须清楚表达强弱：

- 第一拍：红色重音点。
- 其他拍：蓝绿色轻拍点。
- 当前拍：放大、描边或阴影。
- 点内可显示拍号或动作短词。

### 4.3 当前动作

页面中央显示当前动作，不只显示拍号。

示例：

- 第 1 拍：重下扫。
- 第 2 拍：轻下扫。
- 第 3 拍：轻下扫。
- 第 4 拍：轻下扫。

对“下下上上”：

- 第 1 拍：重下扫。
- 第 2 拍：下上。
- 第 3 拍：上扫。
- 第 4 拍：下上。

### 4.4 BPM 控制

- 默认使用当前节奏型 BPM。
- 支持 -5 / +5。
- 范围：40 到 140。
- 正在运行时调整 BPM，应立即重启定时器。

### 4.5 声音

- Preview 使用 Web Audio 合成节拍声。
- 第一拍声音更低或更响，表示重音。
- 节拍声可开关。
- 关闭声音后，视觉节拍继续运行。

App 阶段：

- Expo App 如果暂时不能发出稳定节拍声，应显示“当前平台先显示视觉节拍”的状态。
- 不伪装声音已经播放。

### 4.6 评分与记录

MVP 阶段先使用本地模拟评分：

- 每完成一小节，节奏参考分小幅增加。
- 点击“完成本组”后写入本地练习记录。
- 记录字段至少包含：节奏型、BPM、完成小节、节奏分、时间。

后续接真实麦克风后：

- 用扫弦 onset 与目标拍点偏移计算节奏分。
- 使用 `audio-core` 的 rhythm scoring，不重写评分算法。

## 5. 数据来源

优先使用 `packages/shared`：

- `beginnerRhythmPatterns`
- `mvpPracticeTemplates`
- `getRhythmPatternById`
- `practice-rhythm-down-four`
- `practice-rhythm-down-down-up-up`

Preview 阶段如果已有页面内常量，可先继续使用，但字段含义必须和 shared 保持一致。

## 6. Preview 实现要求

`apps/mobile/dist-web/preview.html` 需要：

- 将当前简易 rhythm sheet 改成正式一屏练习页。
- 支持节奏型切换。
- 支持 BPM -5 / +5。
- 支持节拍声开关。
- 支持开始、暂停、完成本组。
- 当前拍自动推进。
- 完成后更新本地练习记录、首页最近练习和我的页统计。

## 7. App 同步要求

`apps/mobile/App.tsx` 需要：

- 从练琴入口页、节拍器页、首页推荐进入节奏练习。
- 渲染与 Preview 等价的节奏练习结构。
- 使用 shared 的节奏型和练习模板。
- 不在进入页面时请求麦克风权限。
- 完成后写入本地 practice history。

## 8. 验收标准

节奏练习页完成必须满足：

- 从练琴入口页点击“节奏练习”能进入。
- 从节拍器页点击“去练节奏型”能进入。
- 页面显示节奏型选择、当前动作、节拍点、BPM、声音开关和控制按钮。
- 点击开始后拍点按 BPM 自动推进。
- 第一拍明显是重音，其他拍是轻拍。
- BPM -5 / +5 生效。
- 节拍声开关状态可见。
- 点击完成本组后写入最近练习。
- 不请求麦克风权限。
- 不破坏首页、练琴入口页、调音器、节拍器和和弦库。


## 10. 人工测试清单

- 打开 Preview。
- 进入“练琴”。
- 点击“节奏练习”。
- 点击开始，确认当前拍从 1 到 4 自动推进。
- 确认第一拍为重音，其余拍为轻拍。
- 点击 +5，确认 BPM 增加。
- 点击节拍声开关，确认状态变化。
- 切换到“下下上上”，确认当前动作变化。
- 点击完成本组，确认最近练习更新。
- 返回练琴入口页，确认底部 Tab 没有遮挡主要内容。

## 11. 暂不做内容

本阶段暂不做：

- 真实麦克风评分。
- 后台节拍器。
- 复杂自定义节奏编辑器。
- 歌曲级节奏时间轴。
- 云端练习记录。

这些进入后续真实音频、歌曲跟弹和账户数据阶段。
